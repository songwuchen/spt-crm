# SPT-CRM 操作说明书：方案审批 与 严格阶段闸门

> 本说明书基于在真实系统中实际操作（登录 → 配置 → 走通全流程）整理，并记录了操作过程中发现并修复的 BUG 与体验问题。
> 适用对象：系统管理员（配置）、销售/售前（使用）。前置：以管理员登录。

---

## 一、方案审批（让"方案已批准"成为真实审批，而非手动改状态）

### 1.1 管理员：配置「方案审批」策略（一次性）
路径：**系统配置 → 审批策略 → 新增策略**
1. **业务类型** 选「方案审批」。
2. **策略名称** 填写，如「方案技术评审」。
3. **审批人规则 → 添加审批人**：类型选「指定用户」（或「按角色」/「部门领导」），再选具体审批人（如技术总工）。
4. **审批模式**：依次 / 并行（全部通过）/ 任一通过，按需选择。
5.（可选）设置 **SLA(小时)** 与 **升级链**（超时催办 / 自动通过）。
6. 「确定」保存。列表「审批人规则」列会显示**审批人姓名**（如「用户: 管理员」）。

> 未配置策略时，提交评审不会创建审批任务，提交人会收到「未配置审批流程」提醒，此时仍可手动批准作为兜底。

### 1.2 售前：提交方案评审
路径：**商机详情 → 方案 → 新建方案 → 打开方案详情**
1. 编辑方案（概要 / 选型清单 / 风险清单），保存。
2. 点击右上角「**评审中**」按钮 → 确认。
   - 系统据策略**自动发起审批流**，并在方案详情顶部出现蓝色提示条：「方案审批进行中（x/y 节点），请在审批中心处理」，旁附「前往审批中心」。
   - 此时**不再显示手动「已批准」按钮**（防止绕过审批），仅显示「审批中」标签。

### 1.3 审批人：在审批中心处理
路径：**审批中心 → 待我审批**
1. 找到「方案审批: SL-xxxx」，点「通过」（可填意见）或「驳回」。
2. 通过后：方案状态**自动变为「已批准」**，方案详情提示条变为绿色「方案审批已通过」。
3. 驳回后：提示条变红「方案审批已驳回，可修改后重新提交评审」。

### 1.4 效果
方案「已批准」后，商机推进到 **S4 商务谈判** 的闸门「方案已审批」即满足。

## 二、严格阶段闸门（按"状态"卡阶段，例如必须"已签署"合同）

默认闸门只校验"存在"（有合同即可），可按租户配置为校验"状态"。

### 2.1 管理员：配置 S6 需"已签署"合同
路径：**系统配置 → 阶段Gate → 找到 S6 行 → 编辑**
1. 「添加规则」。
2. **规则代码** 填 `HAS_SIGNED_CONTRACT`，**规则名称** 填「已签署合同」。
3. **校验类型** 选「存在关联记录」。
4. **关联实体** 选「合同」。
5. **要求状态（选填）** 选「已签署 (signed)」 —— 这是关键：留空=只要有合同即可；选「已签署」=必须存在已签署的合同。
6. **不通过提示** 填「请先签署合同，再进入交付验收。」
7. 「确定」保存。

> 报价、方案同理可配置（如报价=已发送、方案=已批准）。

### 2.2 使用：推进受闸门约束
- 商机仅有**草稿**合同时点「推进到 S6」→ 被拦截，弹窗提示「缺少 1 项必要条件：已签署合同 / 请先签署合同，再进入交付验收。」
- 在合同详情「签署合同」后，再「推进到 S6」→ 通过，阶段更新为 S6。

## 三、操作过程中发现并修复的 BUG / 体验问题

以下问题均在本次真实操作中暴露并已修复：

1. **【阻断级 BUG】保存阶段 Gate 规则报 422**
   `StageDefinitionUpdate` 把 `gate_rules_json` / `allowed_transitions_json` 标成了 `dict`，但实际是数组 → 保存任何 Gate 规则都 422 失败（这也是此前所有阶段 Gate 规则都为空、从未配置成功的原因）。
   修复：schema 改为 `Optional[Union[dict, list]]`。（`backend/app/domains/admin/schemas.py`）

2. **【阻断级 BUG】可视化 Gate 编辑器无法设置"状态"**
   后端虽支持 `has_related` 的 `status` 过滤，但 `GateRulesEditor` 没有对应输入项，导致"严格闸门"在 UI 上根本无法配置。
   修复：为「存在关联记录」新增「要求状态（选填）」下拉，按实体给出可选状态（合同：已签署/草稿/已终止；报价；方案）。（`frontend/src/components/GateRulesEditor.tsx`）

3. **【体验】审批策略列表"审批人规则"列显示原始 JSON / 用户 UUID**
   管理员看不出到底是谁审批。
   修复：改为友好渲染，按角色/用户解析出**名称**并以标签展示（如「用户: 管理员」「角色: 销售总监」「部门领导」）。（`frontend/src/pages/admin/settings/SettingsPage.tsx`）

4. **【体验】方案提交评审后无任何反馈，且可手动绕过审批**
   方案详情提交评审后看不到审批是否发起、由谁审批；且仍显示手动「已批准」按钮，可绕过审批任务直接置为已批准。
   修复：方案详情顶部新增审批状态提示条（进行中/已通过/已驳回，附「前往审批中心」）；审批进行中时隐藏手动「已批准」按钮并显示「审批中」标签；提交评审弹窗补充说明文案。（`frontend/src/pages/opportunity/SolutionDetail.tsx`）

### 备注（已知、非本次修复范围）
- 阶段定义里 S1–S6 的**名称**（线索确认/需求分析/方案制定/商务谈判/合同签订/交付执行）与商机详情前端展示的阶段名（…/方案报价/…/合同签署/交付验收）措辞不完全一致，属历史文案差异，不影响流转。
- 控制台存在 antd 框架级弃用告警（`destroyOnClose`、静态 `message` 上下文），非业务错误。
