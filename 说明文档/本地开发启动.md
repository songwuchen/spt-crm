# SPT-CRM 本地开发启动指南

本文档说明如何在 Windows / macOS / Linux 上本地启动 SPT-CRM 进行开发调试。

## 一、环境要求

| 工具 | 版本建议 | 用途 |
|------|----------|------|
| Docker Desktop | 最新稳定版 | 运行 PostgreSQL、Redis |
| Node.js | 20.x | 前端开发服务器 |
| Python | 3.12+（推荐 3.12） | 后端 API |
| Git | 任意 | 拉取代码 |

验证环境：

```powershell
docker --version
node --version
python --version
```

## 二、项目结构

```
spt-crm/
├── backend/          # FastAPI 后端
├── frontend/         # React + Vite 前端
├── docker-compose.yml
├── docker-compose.override.yml   # 本地开发端口覆盖（可选）
└── 说明文档/
    └── 本地开发启动.md
```

## 三、首次启动（完整步骤）

### 1. 克隆并进入项目

```powershell
cd e:\ruolin-a\spt-crm
```

### 2. 启动基础依赖（PostgreSQL + Redis）

```powershell
docker compose up -d db redis
```

等待容器健康：

```powershell
docker compose ps
```

应看到 `db` 和 `redis` 状态为 `healthy`。

> **端口说明**：项目根目录已包含 `docker-compose.override.yml`，将 PostgreSQL 映射到宿主机 **5433** 端口，避免与本机已有 PostgreSQL（默认 5432）冲突。若本机 5432 未被占用，可删除该 override 文件，并将 `backend/.env` 中的数据库端口改回 `5432`。

### 3. 配置环境变量

```powershell
# 后端
copy backend\.env.example backend\.env

# 前端
copy frontend\.env.example frontend\.env
```

`backend/.env` 关键配置（使用 5433 端口时）：

```env
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5433/spt_crm
CORS_ORIGINS=http://localhost:5173,http://localhost:5175,http://localhost:3000
AI_PROVIDER=mock
```

`frontend/.env` 默认即可：

```env
VITE_API_BASE_URL=http://localhost:8002
VITE_PORT=5175
```

### 4. 初始化后端

```powershell
cd backend

# 创建并激活虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1        # Windows PowerShell
# source .venv/bin/activate          # macOS / Linux

# 安装依赖
pip install -r requirements.txt

# 数据库迁移
alembic upgrade head

# 写入演示数据（权限、管理员账号、示例客户/项目等）
python -m scripts.seed
```

### 5. 安装前端依赖

```powershell
cd ..\frontend
npm ci
```

### 6. 启动开发服务

**终端 1 — 后端：**

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
$env:REDIS_URL="redis://localhost:6379/0"
uvicorn app.main:app --host 0.0.0.0 --port 8002 --reload
```

**终端 2 — 前端：**

```powershell
cd frontend
npm run dev
```

## 五、日常开发（已初始化后）

每次开发只需：

```powershell
# 1. 确保 Docker 依赖在运行
docker compose up -d db redis

# 2. 启动后端（backend 目录，虚拟环境已激活）
uvicorn app.main:app --host 0.0.0.0 --port 8002 --reload

# 3. 启动前端（frontend 目录）
npm run dev
```

cd backend

cd frontend

## 七、Docker 一键启动（可选）

若不想在本地安装 Python 依赖，可用 Docker 启动全部服务：

```powershell
docker compose up -d --build
```

此方式前端映射到 **80** 端口，后端 **8002**。首次启动需执行迁移和 seed：

```powershell
docker compose run --rm backend alembic upgrade head
docker compose run --rm backend python -m scripts.seed
```

## 八、常见问题

### 1. 连接数据库失败（Windows）

**现象**：`asyncpg.exceptions.ConnectionDoesNotExistError` 或 `ConnectionResetError`

**原因**：本机 5432 端口已被其他 PostgreSQL 占用，连接到了错误的数据库实例。

**解决**：
- 使用项目自带的 `docker-compose.override.yml`（映射 5433）
- 确保 `backend/.env` 中 `DATABASE_URL` 端口为 `5433`
- 重启数据库容器：`docker compose up -d db`

### 2. 前端无法访问 API

确认后端已启动且 `frontend/vite.config.ts` 中代理目标为 `http://localhost:8002`。也可设置环境变量：

```powershell
$env:VITE_API_PROXY="http://localhost:8002"
npm run dev
```

### 3. CORS 报错

检查 `backend/.env` 的 `CORS_ORIGINS` 是否包含前端地址 `http://localhost:5175`。

### 4. Redis 相关功能不可用

本地开发建议设置：

```powershell
$env:REDIS_URL="redis://localhost:6379/0"
```

并确保 `docker compose up -d redis` 已执行。

### 5. 端口被占用

| 端口 | 服务 | 修改方式 |
|------|------|----------|
| 5175 | 前端 | `frontend/.env` 中 `VITE_PORT` |
| 8002 | 后端 | uvicorn `--port` 参数 |
| 5433 | PostgreSQL | `docker-compose.override.yml` + `backend/.env` |
| 6379 | Redis | `docker-compose.yml` |

## 九、技术栈速查

- **后端**：FastAPI + SQLAlchemy (async) + Alembic + PostgreSQL + Redis
- **前端**：React 18 + Vite 6 + Ant Design 5 + TypeScript
- **AI**：默认 `mock` 模式，无需 API Key 即可开发

更多业务与架构说明见项目根目录 `需求文档.md` 及 `docs/` 目录下的产品/运维文档。
