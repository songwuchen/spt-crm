# SPT-CRM 服务器部署指南

本文档说明如何在生产服务器上部署 / 升级 SPT-CRM。

## 一、服务器信息

| 项 | 值 |
|----|----|
| SSH | `ssh songwuchen@192.168.0.39` |
| 建议部署目录 | `/home/songwuchen/spt-crm` |
| Harbor 仓库 | `wmharbor.fourier.net.cn:39011/songwuchen` |
| 前端 HTTP | `http://192.168.0.39:39280` |
| 前端 HTTPS | `https://192.168.0.39:39281`（需证书） |
| PostgreSQL 宿主机端口 | `5439`（容器内 5432） |

> 端口选择说明：与同机 SRM(`390xx/391xx`)、Ruolin(`393xx`)、QMS(`394xx`) 错开；`5438` 已被其他项目占用，故数据库用 `5439`。

镜像地址（由 CI 推送到 Harbor）：

- `wmharbor.fourier.net.cn:39011/songwuchen/spt-crm-backend:latest`
- `wmharbor.fourier.net.cn:39011/songwuchen/spt-crm-frontend:latest`
- `wmharbor.fourier.net.cn:39011/songwuchen/redis:7-alpine`
- `wmharbor.fourier.net.cn:39011/songwuchen/pgvector:pg16`

> 流程：代码合并到 `main` → GitHub Actions 测试并推镜像 → **本机手动**拉镜像并升级（不再自动 SSH 部署）。

## 三、首次部署

### 1. 登录并准备目录

```bash
ssh songwuchen@192.168.0.39
mkdir -p ~/spt-crm && cd ~/spt-crm
```

### 2. 同步部署文件

任选一种方式。

**方式 A：克隆仓库（推荐，后续升级方便）**

```bash
cd ~
git clone git@github.com:songwuchen/spt-crm.git
cd spt-crm
```

**方式 B：只拷贝部署所需文件**

在本地（开发机）执行：

```powershell
scp docker-compose.prod.yml nginx.prod.conf .env.example deploy/deploy.sh songwuchen@192.168.0.39:~/spt-crm/
```

服务器上：

```bash
cd ~/spt-crm
mv docker-compose.prod.yml docker-compose.yml   # 若只用精简目录
chmod +x deploy.sh                               # 精简目录时脚本与 compose 同级
# 若是完整仓库：chmod +x deploy/deploy.sh
```

若出现 `/usr/bin/env: 'bash\r': No such file or directory`，说明脚本带了 Windows 换行（CRLF），在服务器上执行一次：

```bash
sed -i 's/\r$//' deploy.sh
# 或：sed -i 's/\r$//' deploy/deploy.sh
```

然后重新 `./deploy.sh init`。
> 完整仓库布局下，`deploy/deploy.sh` 会自动找到仓库根目录的 `docker-compose.prod.yml`。  
> 精简目录布局下，把 `docker-compose.prod.yml` 改名为 `docker-compose.yml` 并与 `deploy.sh` 放在同一目录。

### 3. 登录 Harbor

```bash
docker login wmharbor.fourier.net.cn:39011
# 输入有 songwuchen 项目 Pull 权限的账号 / 密码（或 Robot Token）
```

### 4. 证书（HTTPS / 39281）

`docker-compose.prod.yml` 会挂载：

```text
/etc/letsencrypt/live/fourier.net.cn/fullchain.pem
/etc/letsencrypt/live/fourier.net.cn/privkey.pem
```

- 若服务器上已有该证书：无需改动。
- 若暂时没有证书：先只通过 **39280** 用 HTTP 访问，或按实际证书路径修改 compose / `nginx.prod.conf` 后再启用 39281。

### 5. 执行初始化

完整仓库：

```bash
cd ~/spt-crm
./deploy/deploy.sh init
```

精简目录（脚本与 compose 同级）：

```bash
cd ~/spt-crm
./deploy.sh init
```

脚本会引导你：

1. 从 `.env.example` 生成 `.env`
2. 设置 `POSTGRES_PASSWORD`、`JWT_SECRET_KEY`、`CORS_ORIGINS`
3. 从 Harbor 拉取镜像
4. 启动数据库 → 迁移 → seed → 启动全部服务

建议按提示配置 `CORS_ORIGINS`，例如：

```env
CORS_ORIGINS=http://192.168.0.39:39280,https://192.168.0.39:39281
```

若使用正式域名，改成域名对应地址（与 `.env.example` 中示例一致）。

### 6. 验证

```bash
./deploy/deploy.sh status
# 或
docker compose -f docker-compose.prod.yml --env-file .env ps
```

浏览器访问：

- http://192.168.0.39:39280

默认账号：

| 用户名 | 密码 |
|--------|------|
| `admin` | `admin123` |

**登录后请立即修改密码。**

## 四、日常升级

CI 已把新镜像推到 Harbor 后，在服务器执行：

```bash
ssh songwuchen@192.168.0.39
cd ~/spt-crm

# 若用 git 管理部署文件，先同步配置变更
git pull

./deploy/deploy.sh upgrade
```

`upgrade` 会：备份数据库 → 拉最新镜像 → 迁移 → 幂等 seed → 重建 backend / worker / reminder / frontend。

快速升级（已自行备份）：

```bash
./deploy/deploy.sh upgrade --skip-backup
```

## 五、常用运维命令

以下在 `~/spt-crm` 下执行（完整仓库用 `./deploy/deploy.sh`）：

```bash
./deploy/deploy.sh status              # 容器状态
./deploy/deploy.sh logs backend        # 跟后端日志
./deploy/deploy.sh logs frontend
./deploy/deploy.sh restart backend     # 重启某服务
./deploy/deploy.sh backup              # 备份数据库到 ./backups/
./deploy/deploy.sh migrate             # 仅跑迁移
./deploy/deploy.sh seed                # 仅跑 seed（幂等）
./deploy/deploy.sh down                # 停容器（保留数据卷）
```

恢复备份：

```bash
./deploy/deploy.sh restore backups/spt_crm-YYYYMMDD-HHMMSS.sql.gz
```

## 七、常见问题

### 1. 拉镜像失败 / denied

```bash
docker login wmharbor.fourier.net.cn:39011
docker pull wmharbor.fourier.net.cn:39011/songwuchen/spt-crm-backend:latest
```

确认 Harbor 项目 `songwuchen` 存在，且账号有 Pull 权限；确认 CI 已成功推送 `latest`。

### 2. frontend 起不来（证书文件不存在）

compose 挂载了 Let’s Encrypt 证书路径。证书缺失时容器可能启动失败。可临时注释掉 `frontend` 服务中的 ssl volume，或准备好证书后再启动。

### 3. 生产机访问不了 Docker Hub

正常现象。本项目的 `db` / `redis` / 应用镜像均走私有 Harbor，**不要**对整个 compose 执行会去拉 Docker Hub 的操作。用脚本的 `upgrade` / `pull` 即可。

### 4. 端口占用

| 端口 | 用途 |
|------|------|
| 39280 | 前端 HTTP |
| 39281 | 前端 HTTPS |
| 5439 | PostgreSQL（宿主机） |

冲突时修改 `docker-compose.prod.yml` 的 `ports` 映射后重新 `up -d`。

### 5. 需要演示数据

首次安装：

```bash
./deploy/deploy.sh init --with-demo
```

已安装环境补演示数据：

```bash
./deploy/deploy.sh seed
# 或进入容器执行带 demo 的 seed（见 deploy.sh 实现）
```
